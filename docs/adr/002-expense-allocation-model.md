# ADR-002: โมเดลการตัดเงิน — แยกรายจ่ายออกจากการจัดสรร (Expense Allocation)

สถานะ: ยอมรับแล้ว (12 ก.ค. 2569)

## บริบท

ใน Excel เดิม (รายจ่าย 2569.xlsx) 1 sheet = 1 ใบเสร็จ ทำหน้าที่เป็นบัญชีแยกประเภท
แต่ในการทำงานจริง **รายจ่าย 1 รายการบางครั้งตัดเงินจากหลายใบเสร็จรวมกัน**
(ยืนยันจากการสัมภาษณ์เจ้าหน้าที่) ซึ่งโครงสร้าง 1 sheet ต่อใบรองรับไม่ได้ —
ต้องแตกรายการด้วยมือและเสี่ยงยอดไม่ตรง

## การตัดสินใจ

แยกเป็น 2 ตาราง:

- `expenses` — รายจ่าย 1 รายการ (เลขที่ส่งออก, วันที่จ่าย, รายการ, ยอดรวม)
- `expense_allocations` — การตัดเงิน: (expense_id, donation_id, amount)
  รายจ่าย 1 รายการมีได้หลายแถว = ตัดจากหลายใบเสร็จ

กติกา (บังคับที่ฐานข้อมูล):

1. ผลรวม allocation ต่อใบเสร็จ ≤ ยอดใบเสร็จ (trigger `allocation_not_exceed`)
2. ผลรวม allocation ต่อรายจ่าย = ยอดรายจ่าย (ตรวจใน RPC `save_expense` — บันทึกแบบ atomic)
3. ยอดคงเหลือ **คำนวณเสมอ** จาก view `donation_balances` ไม่เก็บซ้ำในตาราง
4. เจ้าหน้าที่**เลือกใบเสร็จเองทุกครั้ง** — ไม่มี auto-FIFO (ตามที่ผู้ใช้เลือก)

## เหตุผล

- รองรับกรณีตัดหลายใบโดยไม่ต้องแตกรายจ่ายเป็นหลายรายการ
- ยอดคงเหลือผิดไม่ได้เพราะไม่เคยถูกเก็บ — คำนวณจากข้อมูลจริงเสมอ
- ลบรายจ่าย = คืนเงินให้ใบเสร็จอัตโนมัติ (cascade delete allocations)

## ผลที่ตามมา

- หน้ารายละเอียดใบเสร็จแสดง "ส่วนของใบนี้" เมื่อรายจ่ายตัดหลายใบ
- ข้อมูลเก่าที่ import จากไฟล์รายจ่าย (1 sheet ต่อใบ) จะกลายเป็น
  รายจ่าย 1 รายการต่อ 1 ใบเสมอ (allocation เดียว)
