# ADR-001: Tech Stack — Next.js + Supabase บน Vercel

สถานะ: ยอมรับแล้ว (12 ก.ค. 2569)

## บริบท

ต้องการระบบออนไลน์แทน Excel + ระบบ DDEE เดิม ผู้ใช้เป็นเจ้าหน้าที่ 1-2 คน
ข้อมูลราวปีละ ~1,000 ใบเสร็จ ไม่มีงบค่า hosting และไม่มีคนดูแลเซิร์ฟเวอร์

## ทางเลือกที่พิจารณา

1. **Google Sheets + หน้าเว็บครอบ** — แก้ข้อมูลตรงได้ แต่บังคับความถูกต้องไม่ได้
   (ตัดเงินเกินยอด, ใบเสร็จซ้ำ) และช้าเมื่อข้อมูลโต
2. **เซิร์ฟเวอร์หน่วยงาน** (แบบ DDEE เดิม) — ต้องมีคนดูแล, เป็นเหตุให้ระบบเดิมล้าสมัย
3. **Next.js + Supabase บน Vercel** ← เลือก

## การตัดสินใจ

- **Next.js 16 (App Router) + TypeScript + Tailwind CSS** deploy บน Vercel free tier
- **Supabase** (PostgreSQL + Auth + Row Level Security) free tier
- กราฟใช้ **Recharts**, อ่านไฟล์ Excel ด้วย **SheetJS (xlsx)** ฝั่ง browser

## เหตุผล

- ค่าใช้จ่าย 0 บาท ในขนาดข้อมูลระดับนี้
- ไม่ต้องดูแลเครื่อง — อัปเดตอัตโนมัติผ่าน git push
- ความถูกต้องของข้อมูลบังคับที่ฐานข้อมูล (constraint + trigger) ไม่ใช่ที่หน้าจอ
- PostgreSQL มาตรฐาน — ย้ายออกจาก Supabase ได้ถ้าจำเป็น

## ผลที่ตามมา

- ต้องมีบัญชี Supabase + Vercel (ฟรี) ผูกกับอีเมลของภาควิชา/เจ้าหน้าที่
- Free tier ของ Supabase หยุดโปรเจกต์ที่ไม่มีการใช้งาน 7 วัน — แก้โดยเข้าใช้งานปกติ
  หรืออัปเกรดเมื่อจำเป็น
